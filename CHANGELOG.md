# Changelog

## v0.2.0-rc.1 - 2026-07-11

- Made `agency-chief-of-staff` the canonical explicit and best-effort-implicit entrypoint, while preserving `zhijuan-codex-agency-chief-of-staf` as an explicit-only generated compatibility bundle.
- Made the generated legacy body explicit-only as well as its frontmatter and UI policy, removing the inherited canonical wording that mentioned implicit activation.
- Changed installation to a rollback-protected, manifest-verified two-bundle transaction under an OS advisory lock that crash-releases; source/target ancestor overlap is rejected before any lock or staging mutation, and every reserved backup/staging/failed prefix remains visible and release-blocking even with a malformed or empty suffix. Added `sealed-tree-v1`: exact runtime paths, node types, canonical modes, owner and regular-file link counts are checked with stable `lstat`/nofollow scans, so FIFO/socket/device, internal symlink, hardlink, permission drift, extra files and empty directories cannot masquerade as a current installed pair; `--force` repairs contamination inside a real bundle directory, while a bundle-root symlink is refused.
- Documented that implicit Skill discovery is subject to the host skill context budget and is not guaranteed; deterministic startup uses the canonical explicit slug, never `AGENTS.md` routing injection.
- Upgraded model-smoke receipts to schema V2: macOS RC evidence verifies the OpenAI Team ID signature on Codex and `codex-code-mode-host`, executes frozen private copies, and rechecks source/copies; wrapper, symlink, ambient-PATH helper, and free-text identity evidence remain rejected.
- Added `verified-installed-snapshot`: the runner verifies the OS-home canonical/legacy pair and transaction cleanup before and after, freezes only the allowlisted bundles into each case's clean HOME, rejects snapshot drift and CLI-owned Skill-context overflow, and leaves sibling Skills untouched. RC source provenance now also freezes, rechecks, and receipts the actual imported `scripts/install_skill.py` bytes as `installer_sha256`, so helper replacement cannot preserve an apparently equivalent runner/case/runtime receipt.
- Bound each case to exactly one exec rollout `session_meta` and `turn_context`; model, OpenAI provider, reasoning effort, CLI version, source and cwd must match the request, while mismatched identity values are reduced to fixed errors instead of being copied into receipts.
- Made model smoke explicitly bind a model-supported reasoning effort because `--ignore-user-config` discards personal reasoning settings; the isolated command also enables `multi_agent`, and release eligibility now requires the reasoning binding.
- Classified terminal usage-limit and selected-model-capacity events as explicit external/inconclusive states while keeping suite and release fail closed; only allowlisted transport retries followed by one clean completed turn become recorded warnings instead of misleading behavior-error cascades.
- Aligned the natural full-cycle prompt's final-line terminator with its canonical machine schema, and added a content-free direct-read block-shape witness to model-smoke receipts so changed Codex output envelopes can be diagnosed without persisting artifact text or markers.
- Locked all 25 behavior cases by full canonical JSON SHA-256, including 20 required model-smoke cases. Added model-level negative coverage for first-line worker markers with missing, duplicate, out-of-order, or empty fields, plus a natural, concrete full-cycle write/review request that leaves tool choice to the Skill while mechanically binding its unique child terminal, host delivery, verbatim main-thread adoption, and isolation-unverified disclosure; this natural oracle is explicitly separate from reviewer-owned readback proof. Ordinary focused code fixes and ordinary code reviews remain negative cases so routine engineering work cannot silently expand into Chief-of-Staff orchestration.
- Required positive invocations to load only the selected bundle and match the complete installed `SKILL.md` output. `expected_entrypoint=none` still forbids activation. Default worker packets omit both a `$slug` and a guard request so workers do not recursively start the main Skill; only a host-forced preloaded bundle may contribute one exact first-action guard read. Partial, failed, repeated, wildcard, wrong, or dual-bundle touches fail. Ordinary negatives require exact final text, zero tools, and zero collaboration attempts, including failed or cancelled spawns.
- Bound reviewer delegation to a packet-only artifact-review path: a reviewer may not emit a boot receipt, inherit top-level prose, or open a second turn after any non-empty terminal. Reviewer packets now describe only evidence types and terminal schemas instead of preloading expected artifact text or runtime markers, so a successful terminal must be derived from the current artifact. Strict and natural collaboration oracles continue to reject duplicate terminals, follow-up repair, and unbound reads.
- Made the reviewer packet enumerate only `NO_BLOCKING_FINDINGS` and `BLOCKING_FINDINGS` for `REVIEW_VERDICT`, explicitly rejecting generic `PASS` without embedding an artifact result or marker.
- Accepted fragmented reviewer direct-read output only when every block is a non-empty exact `input_text` pair, an optional leading `Script completed\n` block is removed, and the remaining concatenation byte-matches the complete artifact; raw strings, objects, mixed block types, empty blocks, and any extra whitespace/text remain rejected.
- Made preload validation prompt- and event-grounded: a one-line use announcement preserves prompt negation, rejects opposite actions, completion/result claims and canonical/legacy slug confusion; any earlier tool action fails, and a recovery notice requires exactly one failed simple selected-bundle read before the successful full read. The legacy bundle now refers to the currently selected bundle instead of incorrectly instructing a canonical read.
- Split reviewer evidence by release tier. RC now permits one same-identity/same-turn OpenAI child, binds spawn/start and an exact author/recipient delivery between the child and main terminals, requires a runtime-random marker and exactly one inert-wrapper `/bin/cat` full-artifact action after the final modification, accepts only an exact four-line zero-finding reviewer terminal plus an exact six-line main adoption, and rejects extra/contradictory prose, post-final delivery, premature claims, substring envelopes, extra tools, expressions, intermediate PASS, decoders, generic waits and identity drift. Wait remains a blocking mechanism rather than review truth. Opaque stored spawn prompts contribute no content or marker-nonforwarding proof; receipts distinguish that limitation from the verified behavior chain. A requested `fork_turns:none` cannot prove isolation; receipts explicitly report `stable_supported=false`.
- Bound both collaboration oracles to the current host namespace, call/output/response/delivery parent-turn metadata, current structured `FINAL_ANSWER` delivery envelope, started event id/index/clock and `fork_turns:none` child session shape; strict wait and direct-read causality are also bound. Parent-to-child `agent_message` task transport is now classified separately from child assistant output and must start with one exact visible `NEW_TASK` header plus the bounded encrypted envelope. That shape is syntactic binding, not authentication; ingress must be the first child response item and the rest of the child action/event surface is closed to its allowlist. Missing, duplicate, malformed, unknown, or late transport/action fails closed. Strict progress is accepted only after a matching passive-guard output when such a guard exists. Artifact ordering now uses `max(mtime, ctime)`. Replaced open-ended intermediate-message semantics with exact boot and ordered `MAIN_PROGRESS:*` / `REVIEW_PROGRESS:*` allowlists. Clean-eval rollout response, raw event, and exec JSON message surfaces must each expose one unique terminal and are validated independently with exact current schemas, phase and causal-time checks, closing missing-surface, mixed-content, cross-turn-delivery and unmirrored-event bypasses while allowing legitimate cross-surface mirrors.
- Replaced one-pass rollout extraction with two root-anchored mutation-stable scans that compare directory state, first/last enumeration, file descriptors, parent-name bindings, bytes and metadata manifests. Added a dedicated full `.git` manifest that includes ignored names, `__pycache__`, empty directories, permissions/ownership, inode/time metadata and content hashes instead of reusing the Skill-install manifest.
- Made write-case file integrity independent of Git ignore/user config by enumerating ignored/untracked paths and comparing `.git` metadata before/after; also added root-name-to-inode revalidation for the rollout `sessions` directory.
- Scans every bounded raw main/child/unrelated rollout both lexically and after JSON decoding before RC handling can pass. CODEX_HOME is anchored to its pre-run inode and scanned twice through component-wise nofollow dirfds; only a mutation-stable snapshot shared by identity, review and auth checks is accepted.
- Moved each model-smoke project parent away from its Skill HOME and auth-bearing CODEX_HOME, and made that private-state separation an explicit RC receipt gate; the same-user process remains outside any secret-boundary claim.
- Replaced free-form pre-read announcements with deterministic canonical/legacy templates, strengthened legacy read-before-boot metadata, and allowed task-required post-boot reads of only the selected bundle while preserving fail-closed opposite-bundle detection.
- Split RC artifact evidence from operational credential safety. Primary or declared-dedicated auth can support a reviewed-source RC only when the full artifact and auth-handling gates pass; neither class proves low privilege or isolation, and stable remains closed without authoritative workload identity and zero untested capabilities.
- Added privileged-Bash `scripts/model_smoke.sh`, receipt-tier enforcement (`rc` / `stable`), auth-file run binding, exact-value rollout-evidence scanning, and a nested release decision that cannot be satisfied by process exit 0 alone.
- Added a high-confidence secret scanner and a private checked `rg` copy; runner-owned Git operations now use a trusted absolute helper and sanitized environment.
- Hardened offline gates against ambient `PATH`, `BASH_ENV`, `PYTHONPATH`, user-site, and ignored-but-force-tracked publication bypasses; fixed absolute helpers, Bash privileged mode, isolated Python startup flags, symlink-launcher rejection, and tracked secret/symlink regression tests are now required.
- Added POSIX process-group timeout cleanup and atomically promoted model-smoke output staging; interrupted or exceptional runs cannot leave the requested final receipt directory, and independent `setsid` escapees remain explicitly outside the guarantee.
- Changed event JSONL auth scanning from raw byte-pattern replacement to recursive semantic JSON-string redaction, covering quotes, backslashes, newlines, nested values, malformed-line fail-closed preservation, and explicit rejection of valid JSON records that are not event objects, including records after a terminal event.
- Added a content-addressed canonical/legacy package-source revision to installer receipts and validated the public canonical/worker prompt examples as part of the offline contract.
- Rewrote the 1,560-line role-and-receipt framework into an under-200-line outcome-owner workflow: goal, research, minimal plan, execution, verification, independent review, repair, and delivery.
- Made the main task a valid execution surface and limited delegation to work that benefits from parallelism, isolation, or an independent view.
- Replaced the fixed 16-role organization with dynamic research/execution/review subagent responsibilities and host-selected current models.
- Removed every active `AGENTS.md` routing snippet, installer flag, template, and validation dependency; fresh installs leave it untouched, while an explicit `--force` upgrade reports removed obsolete package-internal guidance only after cleanup, otherwise returning pending guidance plus residual backup paths.
- Replaced full-repository copying with a nine-file runtime allowlist and staged replacement with rollback.
- Replaced grep-heavy self-referential release gates with explicit package/contract checks, installer unit tests, and a real `codex exec` model-smoke runner.
- Split specialized guidance into four short on-demand references for real threads, delivery review, long-running work, and historical audits.
- Marked v0.1.x validation receipts as historical evidence rather than current model-behavior proof.
- Hardened the installer against source/target symlinks, pair/source drift before promotion, active-lock conflicts, and post-commit backup-cleanup failures; cleanup failures remain visible on later idempotent calls, with an independent runtime allowlist in the package validator.
- Hardened model smoke against unsafe case ids, artifact traversal, dangerous sandboxes, environment-secret inheritance, plugin/app contamination, partial-run overclaims, incomplete cold-review events, extra dirty files, and exact auth-value output leakage; shell tools now receive an explicit minimal PATH while all parent environment inheritance remains disabled.
- Made model smoke require an explicit auth path and risk acknowledgement; documented that same-user temp auth is not a secret boundary and untrusted diffs need a disposable user/container plus dedicated low-privilege credentials.

## v0.1.7 - 2026-07-06

- Clarified that real customer-project injection, DIR live-user acceptance, and ADCO client/PPT-ready/domain acceptance are post-release dogfood boundaries, while customer-facing client-ready claims still require `DOMAIN_DELIVERABLE_RECEIPT`.
- Added project-state convergence receipts and field-level `release_receipt` gates so release status claims must match explicit receipt fields.
- Clarified README release status wording so the package is described as local-hardened release-candidate quality instead of over-claiming open-source-ready publication.
- Added a required Chinese `THREAD_DISPATCH_RECEIPT` dispatch summary card so worker dispatches are readable before any machine YAML appears.
- Added Chinese-first compact visible output rules so light `COS_BOOT_RECEIPT` responses stay readable while machine receipts remain available for dispatch, heartbeat, and release evidence.
- Added the same Chinese-first output rule to the project/global `AGENTS.md` routing shim and validation gates so cross-project COS starts do not regress to English-heavy receipts.
- Added explicit installer support for project/global `AGENTS.md` routing shims without making the default install silently modify user rules.
- Hardened the dispatch handshake so `thread_dispatch_decision: dispatch` must immediately converge to `THREAD_DISPATCH_RECEIPT` or `TOOL_BLOCKED`.
- Added bounded worker receipt polling so dispatched workers that do not return receipts are marked `thread_not_converged` and rescued instead of passively waited on.
- Blocked the live-test failure mode where COS falls back to current-worktree implementation after both the original worker and rescue worker fail to return receipts.
- Documented that Codex automation heartbeats execute their configured prompt and only start this Skill when the prompt or AGENTS routing explicitly invokes it.
- Added poll pacing and startup grace so complex workers are not killed by three rapid readbacks before they have time to produce a receipt.
- Added domain-deliverable gates and `DOMAIN_DELIVERABLE_RECEIPT` validation so creative, storyboard, proposal, research, copy, story, execution, and planning work cannot be called client-ready from process receipts alone.
- Tightened domain-deliverable validation so `verdict: PASS` requires passing domain gates plus `cold_reviewed_and_domain_reviewed`; all-FAIL gate receipts are now blocked.
- Tightened activation validation so UUID-only dispatch self-reports are invalid while legitimate `pending_worktree_id` + `dispatch_pending` receipts remain valid.
- Added release convergence gates with `max_review_waves`, `max_parallel_reviewers_per_deliverable`, required `add_review_wave_reason`, and stuck-review bounded rescue validation.
- Hardened worker receipt identity so role-specific worker packets must include the worker's own Codex `thread_id`; receipts that copy `source_thread_id` or the main thread id are now rejected as `invalid_worker_thread_id`.
- Hardened worker dispatch prompts so the Chief-of-Staff must inject the worker's actual `thread_id` into the worker prompt and record `worker_prompt_identity_contract: included`; dispatch receipts missing this proof are blocked.
- Hardened natural heartbeat acceptance so reviewers must return `NOT_DUE` before `next_natural_due_at_local` instead of misclassifying user-triggered in-progress turns as heartbeat failures.
- Hardened missing-cwd worker handling so a worker cannot self-create or re-checkout its missing isolated worktree and then claim adopted execution evidence.
- Added `validation/release_receipt.json` plus `scripts/validate_release_receipt.py` so dispatch, adoption/rejection, cleanup, and review verdicts converge into one machine-readable release artifact.
- Expanded implicit routing metadata for release readiness, public repository publishing, reusable Skill hardening, and multi-file reliability validation.
- Tightened black-box complex prompts so realistic complex tasks must require dispatch-or-TOOL_BLOCKED without leaking thread/receipt wording.
- Added black-box complex-task activation prompts that avoid `$skill`, COS, thread, receipt, and cleanup giveaway terms.
- Added activation fixture coverage for COS main-thread over-execution so direct implementation claims are not accepted as routed worker evidence.
- Clarified that Skill descriptions improve selection probability, while project `AGENTS.md` routing is required to force default Chief-of-Staff behavior.
- Added a local historical-thread audit script and fixture to detect prior Chief-of-Staff activation, dispatch, pending worktree, non-converged review, title/readback, and cross-project routing failure modes.
- Documented the historical evidence hierarchy so old thread titles, worker self-report, `pendingWorktreeId`, and `thread_not_converged` cannot be mistaken for completion evidence.
