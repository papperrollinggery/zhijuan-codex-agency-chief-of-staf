# Code of Conduct

## Our standard

We want this project to be a useful, evidence-driven place to collaborate. Participants should:

- discuss ideas and code without attacking people;
- give specific, actionable feedback and distinguish facts from assumptions;
- respect different levels of experience, language, identity, and background;
- avoid publishing private information, credentials, private paths, or unredacted logs;
- accept correction when a claim is not supported by current evidence.

Harassment, threats, discriminatory language, sexualized attention, deliberate intimidation, doxxing, and sustained disruption are not acceptable.

## Scope

This standard applies to repository issues, pull requests, reviews, discussions, release conversations, and other project spaces. It also applies when someone is representing the project in a public venue.

## Reporting and enforcement

For concerns that are safe to discuss publicly, open an issue with only the information needed to understand the conduct problem. Do not publish private or identifying details.

For threats, harassment, doxxing, or concerns that should not be public, use [GitHub's Report Abuse process](https://support.github.com/contact/report-abuse). Security vulnerabilities belong in the repository's [private security reporting flow](SECURITY.md), not in a public conduct report.

Maintainers may edit or remove harmful content, lock conversations, close contributions, or temporarily or permanently restrict participation. Enforcement should be proportionate, document only what can safely be public, and avoid implying that an investigation established facts it did not verify.

## Attribution

This project-specific policy is informed by common open-source community standards and GitHub's community moderation guidance. It is not a claim of affiliation with another project or standards body.
